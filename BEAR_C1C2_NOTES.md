## 2. 数据获取

- `LIMIT = 2000` 根 1H K线（约83天）。**Bybit V5 kline 单次请求上限是 1000 根**（交易所
  服务端硬限制，ccxt 不会自动翻页），所以 `fetch_ohlcv_paginated()` 用 `since` 分页：
  从"现在 − 2000小时"开始，每批 1000 根往后接，最后取最新的 2000 根。
- `ohlcv[-1]` 是还没走完的当前小时K线。
- **图表永远画到最新一根蜡烛**（用户明确要求）。
- 短时间内反复跑会触发 Bybit `10006 Rate Limit`，等 60 秒再试。

## 3. 核心概念：滚动 Base

空头 VCP：下跌趋势中"反弹-收缩-再反弹-收缩更小"，每次反弹（C1, C2, C3...）一浪比一浪弱，
最后**跌破 C1 低点 = 突破完成**，从 C1 到突破点的整个区域封为一个 **Base**。突破之后继续
找下一组 C1/C2... 形成 Base 2、Base 3...，整个下跌趋势就是一串阶梯（每个 Base 一级台阶）。

层级关系一句话：**Reset 管 Base 的编号周期，MA50 触碰管 C1 的资格，跌破 C1 低点管 Base 的封口。**

### 4.1 Reset

```
MA150 金叉 MA200（从下往上穿）之后，价格第一次重新回到空头排列 (MA200>MA150>MA50)
```

- Reset 的作用是 **Base 编号清零**：每个 Reset 周期内 Base 从 1 开始数。
  Reset **不是** C1 的成立条件（Base 1 封口后找下一个 C1 不需要新的 Reset）。

### 4.2 C1 的成立条件（四条，缺一不可）

1. **低点**：从搜索起点（Reset 点，或上一个 Base 的突破K线）逐根跟踪最低 low，
   出现新低就把低点后移，且下面两个计数**全部重新数**；
2. **连续3根确认**：低点之后出现连续 3 根"阳线或长影线"（`is_c_candle_bear`，
   跟 C2 用同一标准，中间夹一根不合格蜡烛就重新数）；
3. **碰到 MA50**：低点之后某根K线 `当根最高价 ≥ 当根 MA50`（影线碰到即算，不要求收盘站上）。
   **前提（2026-07-10 定稿，SIREN案例）：C1 低点那根的收盘价必须在 MA50 下方**——
   价格从均线下面弹上来碰它才叫"碰到"；价格本来就骑在 MA50 上方时 touch 条件形同虚设，
   会产生假 C1（如刚 Reset 完均线纠缠时的杂波）。
   **实测方向性（2026-07-20，9币×2000根1H，各自最近一次 Reset 之后）**：空头 68 个
   低点候选里 17 个不合格（25%），其中 4 次真的拦下了已凑齐3连的 C1（BTC 07-18 维持
   45根、SUI 07-10 41根、LINK 07-11 23根、SOL 07-10 11根）；**多头镜像侧 113 个高点
   候选只有 2 个不合格（1.8%），都紧贴均线（≤0.3%）、只活1~2根、都没凑齐3连——多头
   方向这条基本是空转的保险丝**。机制：空头 Reset 触发时价格常常还骑在 MA50 上方
   （BTC 那段 reset 后 92% 的K线收在均线之上），低点就锚在上方；多头的新高天然出现
   在上冲顶部（=MA50上方），没有对应窗口。⚠️ 别因"多头用不上"就把它从多头引擎删掉：
   镜像对称是这套代码的地基，成本也只是一个布尔判断。
   **触碰必须发生在 C1 自己的反弹段内（低点~高点之间）**（HIGH案例，2026-07-10）：
   不能拿高点之后、下一个C领地里某根上影线的触碰给 C1 背书——HIGH 的 C1 反弹顶
   离 MA50 还差得远，却靠 20 根之后 C2 区域里的一次触碰混过了资格审查。
   2 和 3 顺序不限，都凑齐才算成立；
4. **高点**：`find_leg_high()` —— 区域截止于"出现跌破低点的新低"或"**MA150 上穿 MA200**"
   （先到哪个算哪个），区域内最高价 = C1 高点。
   **翻转判定只看慢速对 150×200（2026-07-10 定稿）**：MA50 与 MA150 谁上谁下
   不是结构事件（用户："reset 不是 150 穿 50 或 50 穿 150"——同理不截断反弹区域；
   SIREN 07-06 的合格 C1 曾被 50×150 穿越在低点后第一根就腰斩区域而误杀）。**高点候选从低点当根算起**：低点那根
   K线自己的上影线也算（SIREN案例，用户："低点应该画到同一根的高点，因为这个区域里
   没有比这个更高的高点"）。真实反弹守卫不受影响：低点之后必须至少有一根
   没跌破低点的K线，否则整段不成立。

### 4.3 C2~C6 的成立条件

1. **搜索起点**：前一个 C 的**高点+1**（用户原始规则："C2是C1高点之后继续往后找"）。
   高点之后的回调K线，哪怕参与过前一个C的确认，也属于下一个C的搜索范围
   （SLX 案例：C1高点884，885那根深下影线就是C2的低点，不能因为它是确认蜡烛就跳过）；
2. **触发**：连续 3 根阳线/长影线（`find_c_pattern`）。
   **一组3连确认只能被消耗一次（2026-07-16 定稿，1000FLOKI C3 案例）**：
   C 成立、取代、无效跳过——处理完任何候选之后，下一次触发搜索从该确认点之后开始
   （scan_from = confirm_idx+1）。C1 的确认组同样被 C1 消耗：链循环的 scan_from
   初始化为 `c1['box_end_idx']+1`，C2 要找新的3连；
3. **低点**：搜索起点到触发点这段里的最低 low。两条位置检查（2026-07-16 精修）：
   a) **不能落在3连的第二/三根**（确认蜡烛自己的下影线冒充低点——H/USDT 假C3案例：
      低点是确认组最后一根阴线的下影线，"反弹"只活了一根就被击穿）。
      **可以是3连的第一根**（跌出新低的长下影反转蜡烛自己开启3连——SIREN Base3
      1928案例，用户："1928~1930 这一段就是啊"）；
   b) **低点那根必须"主动创新低"**（low 低于它前一根的 low）：新回调的开始一定是
      砸出新低的那根。SLX 885 ✓（从 C1 高点砸 7.5% 创新低，合法开启 C2）；
      SIREN C7 的 1893 ✗（low 0.02800 > 前一根 0.02795，只是段边界上碰巧的"段内最低"，
      不是新回调——它其实是 C6 反弹的一部分）。
   ⚠️ 实现注意：作废跳过时只推进**触发搜索位置**（trigger_from/scan_from），
   段的起点（找低点的范围）**必须固定在前一个C的高点+1**——不然段会被削到只剩
   确认组3根、低点永远落在确认组里、永远无效，把合法的C全部误杀（SLX 的 C2 就被
   这样误杀过一轮）；
4. **高点**：低点这波反弹的最高点（`find_leg_high`，同 4.2 第 4 条，候选从低点当根算起）。
   **同根 C 合法（2026-07-16 确认 07-09 规则原意）**："接下来区域内的K线高点都没高过
   低点当根的高点时，取当根上影线——区域内没有比这更高的点"（SIREN Base2 的
   C2 +10.6% 和 C6 +2.0% 都是同根 C，用户确认"C6 本来就是对的2%"）。
   前提是 C 的其他成立条件（3连确认等）都符合。
   ⚠️ **如果低点之后下一根就跌破了低点（没有真实反弹区域）→ 这个确认点作废跳过**
   （见 6.4 踩坑）——"同根合法"与"必须有真实反弹区域"不冲突：前者是高点取哪里，
   后者是区域是否存在；
5. 低点跌破**上一段**的低点**不会**终止链条（旧规则已废除）——只有跌破 **C1** 低点才有意义。

### 4.4 取代规则（用户原话 + 级联补充）

> "一旦超过前一个C的高点，那么C1的高点就是C2的高点，这样就删除掉C2，并且重新开始找。"
> "如果下一个C的%大过上一个C 也会取代上一个C。"

- 候选段 `高点 ≥ 参考段高点` **或** `% ≥ 参考段%` → 原地更新参考段（延伸），不新增；
- **级联取代**（2026-07-09 加）：一个 C 被延伸撑大后 % 反超它**前面**的 C → 把前面那个吞掉
  （`del chain[-2]`，循环检查）。C1 不参与被吞（它的低点是整个 Base 的锚）。

### 4.5 突破 / 封口

- C1 成立后，**任何一根K线的 low 跌破 C1 低点的瞬间**就算突破完成（不用等下一个C确认），
  那根K线就是 Base 的右边界；
- **必须至少有一个合法的 C2**：只有 C1 就被打穿的不算 Base（用户："没错很快打穿没有C2的
  都不算base"），直接从突破点继续找下一个 C1；
- **超过 6 个 C 不作废（2026-07-16 定稿，取代旧的"整段作废"）**：结构继续跟踪
  （C7、C8…按同样规则形成/取代），跌破 C1 低点照样封 Base、照样画，只是 Base 打上
  `weak` 标记——"这个C的强度不强，他还是最后能突破并且成立一个base"。
  图上标 `Base N (weak)`，终端输出标 `[弱Base: 超过6个C]`。
  ⚠️ 弱 Base 也计入扫描器的"已完成 Base 数"分级（要不要打折用户还没拍板）；
- 数据走完还没被跌破的 = 未封口结构（活结构），单币图上**画虚线框**（2026-07-14 起，
  见 4.6），终端打印 `[OPEN]`——这是扫描器推送的主体。

### 4.6 画图规则

- **C 段的框**：左边界 = 低点，右边界 = `max(确认完成点, 高点)`——把确认用的3根蜡烛罩进去，
  避免出现 2 根宽的窄条（用户："画一个范围吧，不然2根的话会混乱"）。纵向 = 低点价到高点价。
  标签是框顶居中的紧凑小字 `C2 +3.3%`；
  **相邻的C框不能重叠**（用户，DRIFT案例）：右边界再裁剪到"下一个C的左边界−1"，
  检测数值不动、纯画法裁剪；
- **Base 的框**：固定金色（`#FFD700`，边 `#B8860B`），纵向 = C1低点到C1高点，
  横向 = C1低点K线到突破K线，罩在所有 C 段小框外面，左下角标 `Base N`
  （弱 Base 标 `Base N (weak)`）；
- **活结构（未封口）也画（2026-07-14 用户定）**：C 框用**虚线**、更淡的填充
  （`draw_price_range(dashed=True)`，alpha 0.12），并在 C1 低点画金色虚线触发线
  `C1 low xxx`（跌破=突破）。标题加 `+ OPEN Cn`；
- **FLIP 线（2026-07-14 用户定，"要明显容易看见"）**：活结构 C1 之后慢速对第一次翻转
  （空头=MA150上穿MA200）的位置画**洋红粗竖线**（#FF00FF，lw2.5）+ 粗体 `FLIP` 标签
  ——一眼看出活结构在哪里失效。终端打印 `[FLIP] 慢速对翻转于 idx=…`。
  只在活结构存在且发生翻转时画；
- MA 颜色：MA50 灰 `#808080`、**MA150 深青 `#008B8B`**（2026-07-14 从亮青 #00FFFF 换的，
  白底上更清楚）、MA200 红 `#FF0000`；
- C 段颜色循环 `['#1f77b4','#9467bd','#d62728','#ff7f0e','#2ca02c']`，每个 Base 内部从 C1 重新数；
- 输出四类图：`FULL`（Reset前210根→最新）、`ZOOM`（第一个结构→最新）、
  `BASE1`/`BASE2`/...（每个 Base 的特写，前20后40根上下文）；
- 画图时必须 `set_xlim/set_ylim` 锁定窗口范围，并跳过窗口外的结构框——否则窗口外的
  Rectangle 会把坐标轴撑出大片空白（踩过坑）。

### 4.7 筛选管线与分级（scan_open_c1c2.py / scan_open_c1c2_bull.py）

**管线**（顺序执行，前一关不过直接 out，省请求省计算）：
1. **日线 Stage 关（2026-07-18 起）**：空头只收 **Stage 4**、多头只收 **Stage 2**
   （±0.3%平带三态斜率，只用已收盘日线，见 4.8；替代旧的二元 is_ma150_falling/rising，
   两个函数已从扫描器删除，main.py / run_once.py 里的原版不动）。
   **新币豁免**：上市不足200天（日线不够200根）跳过本关，
   只看1H（跟多头 main.py 的 new-listing 豁免同一先例，用户确认）；
2. **流动性（2026-07-16 用户定）**：日均成交额 =
   `最近50根1H平均量（=TradingView Volume MA50，用户TV上就设50）× 现价 × 24` ≥ **$1M**

3. **1H 空头排列**：最新K线 MA200>MA150>MA50（用户："一看不是空头排列就可以直接out，
   不需要去看多少个base"）；
4. 全过的进**图池**，图池内跑 `find_bases()` 检查**活结构**
   （C1+至少一个C2，还没跌破C1低点）——没有活结构的不上榜；
5. **C1 健康度（2026-07-16 用户定）**：活结构 C1 反弹/回调 < **5%**（`MIN_C1_PCT`）=

## 6. 踩坑记录（为什么规则长这样）

### 6.1 下一个C的搜索起点：反复横跳过，最终定为"高点+1"
- 最早：高点+1（用户原始规则）。在旧的"段内取高点"提取方式下产生微型C重叠灾难
  （每隔两三根就确认出一个C、6段上限被灌满、合格结构被冤枉作废）；
- 中途改成"确认点之后"治住了微型C，但 SLX 案例暴露它会把"高点之后、参与过确认的
  回调K线"跳过，导致 C2 低点错误；
- **最终（07-09定稿）：回到高点+1**。微型C问题由三道闸兜住：高点用 find_leg_high
  延伸提取（6.2）、低点之后无真实反弹则作废（6.4）、取代+级联取代（4.4）。

### 6.2 C的高点不能被确认点截断
低点定下来之后，反弹的高点要用 `find_leg_high` 一直找到"更低的新低"为止，可以越过确认点。
只在"起点~确认点"小段里取高点会造出低点高点同一根的垃圾段。

### 6.3 高点必须在低点之后（不能回头画）
"每一个C都是要从低点画到低点**之后**的高点"。早期版本段内低点高点谁先谁后无所谓
（07-03 笔记如此记载），**07-09 用户明确推翻**：高点不能回头去框低点之前的K线。
再精修（SIREN案例）：高点候选从低点**当根**算起——低点那根自己的上影线算数，
"区域里没有比它更高的高点就画到同一根的高点"；但低点之前的K线永远不算。

### 6.4 假C2：单蜡烛 + 长影线阴线凑数
DOGE idx 1582~1605 曾冒出一个假 Base：C1 对，但"C2"是三根长影线**阴线**在纯下跌里凑出
确认、低点之后没有任何反弹、兜底逻辑用低点当根的上影线充当高点。修复 = 4.3 第 4 条的
"没有真实反弹就作废"。
⚠️ 别和"同根C合法"（4.3 第 4 条，2026-07-16）搞混：这里的病根是**反弹区域根本不存在**
（下一根就跌破低点）；同根C的反弹区域存在，只是区域内最高点恰好是低点当根的上影线。

**长影线的定义（2026-07-09 用户确认为最终版，不要改）**：
```
body = |收盘 - 开盘|          # 实体
wick = (最高 - 最低) - body   # 上下影线总长（不分方向、不看颜色）
长影线 = wick > body          # 实体为0（十字星）无条件算
```
曾考虑过收紧成"长下影线"，用户明确表示要的就是上面这个算法——阴线长上影也算合格蜡烛，
由"必须有真实反弹"规则兜住假确认，不动蜡烛定义本身。

### 6.5 更早的坑（07-03 版，简记）
- C1 高点区域边界 = "新低或均线翻转"，这条从未变过；
- `find_c_pattern` 触发本身就是有效信号，不要用 region_end 之类更远的边界盖过它；
- 低点/高点要在同一段里一起取、不要两步分开找——07-09 版在此基础上加了"高点在低点之后"
  和"可越过确认点"两条修正。

### 6.7 同根C的领地 + "穿过MA50即成立"（2026-07-19 / 07-20 用户拍板）

**(a) 同根C例外（2026-07-19，ESPORTS 案例）**：下一个C的段起点由"前一个C的高点+1"
改为"前一个C的框结束+1"，**仅当前一个C是同根C（high_idx == low_idx）时**生效：
```
seg_start = ref['box_end_idx'] + 1 if ref['high_idx'] == ref['low_idx'] else ref['high_idx'] + 1
```
病根：普通C的高点在低点之后若干根，"高点+1"天然把确认组隔在段外；同根C退化之后
"高点+1" == "低点当根+1"，于是给这个同根C凑确认的那3根K线被划进下一个C的低点搜索
范围，在确认组里挖出假低点、冒出不该存在的C。定稿口径：**确认组是同根C的领地**。
- 只影响下一个C的低点搜索范围与3连触发下限；**突破检查不受影响**，仍从高点+1扫起
  （该封口就封口，不会被延迟）；
- 别和 6.6 第5条已回滚的"领地=region_end"混淆：那版领地膨胀到几十根外吞掉 C3~C6，
  这版领地只到确认组末根为止；
- **不能简化成"所有C一律 boxEnd+1"（2026-07-20 实测否决）**：11币×2000根1H、36 组相邻C
  里有 2 组会被改坏（5.6%）——SLX Base1 C1→C2（C1高点884、框尾885、C2低点正好就是885，
  即 4.3.1 记的那个锚点）和 SOL Base1 C1→C2（锚点1761、框尾1767、C2低点1765）。多数情况
  下反弹高点落在确认组之后，`boxEnd == 高点`，两种写法等价，所以只有 5.6% 分岔。
  ⚠️ **这个结论对"3连从哪根起算"敏感**：若改成锚点当根算第一根（见下一条），SLX 的框尾
  变成 884 == 高点，该案例消失，只剩 SOL 一例（2.8%）——结论仍成立但证据变薄。
  **关键：SLX/SOL 与 ESPORTS 都满足 `boxEnd > 锚点`，却要求相反的处理，唯一能区分
  它们的就是"高低点是否同根"——这个条件判断消不掉。**
  理解方式：普通C的高低点分开，"高点+1"天然把段起点推到离低点足够远的位置；同根C
  时高点==低点，"高点+1"退化成"低点+1"，下一个C的低点搜索从低点的下一根就开始，把整
  个确认组吞进去。例外补的是这段塌掉的距离，不是给所有C加一道更严的闸；
- **"3连是否包含锚点当根"——口径待用户拍板（2026-07-20）**：现在代码是**锚点那根不计入**
  （新低/新高分支里 `streak = 0` 后 `continue`，走不到 `is_c_candle`），4.2 第2条也写的是
  "低点**之后**出现连续3根"。但用户 2026-07-20 多次表达的意图是**锚点算第一根**。在内存
  里把 `streak = 0` 改成 `streak = 1 if is_c_candle(当根) else 0`，跑 11币×2方向：**21 个
  完全一致，1 个有变化**——唯一差异是 SLX Base1 的 C1 框尾 885→884；所有 C 的高点/低点/
  百分比、Base 个数、突破点全部不变。结论：**近乎零风险的改动，改不改是口径问题不是
  风险问题**。脚本 `scratchpad/ab_anchor_counts.py`（源码级打补丁后 exec，不动仓库文件）。
  ⚠️ 记录的 idx 会随数据窗口漂移（同一天 SOL 就从 1764 漂到 1761），复现要用时间戳或
  TRUNCATE_END 固定；
- ⚠️ 2026-07-20 状态：只落在两个 Pine 文件里（`Bear/Bull VCP C1C2 Bases.md`），
  `plot_c1c2_bear_doge.py` / `plot_c1c2_bull.py` 都还没搬——TV 与 Python 在同根C处
  可能有差异，以 Pine 为准。

**(b) C的资格只看"MA50交互 + 3连确认"（2026-07-20 用户澄清）**：**任何一根穿过 MA50 的
蜡烛都算，与颜色无关**——判定就是 4.2.3 的两个条件落在同一根上：`当根 high ≥ MA50`
且 `当根 close < MA50`。用户："不只是阴线穿过，具体来说，只要蜡烛穿过就算了"。
两种常见形态都合法：① 开在 MA50 上方、收在下方的阴线（high = 开盘价）；② 上影线刺穿
MA50、收盘落回下方的阳线（high = 上影线顶）。再加上"连续3根符合C的蜡烛"，C1 即在该
区域成立（用户："只要穿过 MA50，然后又是连续3根符合C的蜡烛，就可以在这个区域成立C1"）。
同根C出现在 C2~C6 位置同理合法："毕竟它符合了连续3根符合C的蜡烛"。
- 推论：**pct 是C的标注，不是资格门槛**（它仍参与 4.4 的取代/级联取代比较，但没有任何
  pct 阈值决定一个C是否成立）。高点取区域内最高价（含低点当根，见 6.3），不要求它在
  时间上晚于低点；同根C的 pct 可能等于那根的全幅跌幅，这是接受的口径；
- 已否决方案（勿再引入）："同根C时高点那根必须是阳线"（想借阳线 high==收盘 保证高点
  时间上晚于低点）——会动到 SIREN C2/C6，用户明确不要；
- 措辞澄清：代码注释里的"取当根上影线"（`plot_c1c2_bear_doge.py:104`）是口语说法，
  实际取的是该根的 **high 字段**（`max(c[2] for c in region)`）。没有上影线时
  high = max(开盘, 收盘)，照样是确定值，无需特殊处理。
